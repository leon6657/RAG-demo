# AI Agent 入门

## 什么是 Agent

AI Agent（智能体）是一种能够自主感知环境、制定计划、使用工具并执行行动的 AI 系统。与传统的 LLM 链式调用不同，Agent 可以动态决定调用哪个工具、何时调用、以及如何解释工具返回的结果。

## ReAct 模式

ReAct（Reasoning + Acting）是 Agent 最核心的设计模式，由 Yao 等人在 2022 年提出。它将推理和行动交替进行：

1. **思考（Thought）**：模型分析当前状态，决定下一步做什么
2. **行动（Action）**：执行一个具体动作，如调用 API、查询知识库、运行代码
3. **观察（Observation）**：接收行动返回的结果
4. **重复**：基于新的观察继续思考，直到得出最终答案

这种循环使得 Agent 能够处理需要多步推理的复杂任务。

## 工具调用

Agent 的能力很大程度上取决于它能够调用的工具。常见的工具类型包括：

- **知识检索工具**：如 RAG 系统的向量搜索
- **计算工具**：如数学计算器、代码解释器
- **外部 API**：如天气查询、数据库查询
- **文件操作**：如读写文件、网络爬虫

每个工具通常包含名称、描述、输入参数和调用方式等元信息，Agent 根据当前需求选择合适的工具。

## Agent vs Chain

在 LangChain 等框架中，Chain（链）是确定性执行的——每一步是预先定义好的。而 Agent 是动态执行的——模型决定下一步做什么。前者适合流程固定的任务，后者适合需要灵活决策的复杂场景。

## LangGraph 简介

LangGraph 是 LangChain 团队推出的状态化管理框架，将 Agent 的 ReAct 循环建模为有向图（StateGraph）。节点（Node）表示计算步骤，边（Edge）表示控制流。LangGraph 的核心优势是显式的状态管理、条件路由和循环支持，适合构建更复杂的多 Agent 系统。
